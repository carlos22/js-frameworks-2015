var Greeting = React.createClass({
    getInitialState: function () {
        return {name: 'User'};
    },
    onChange: function (e) {
        this.setState({name: e.target.value});
    },
    render: function () {
        return (
            <div>
                <h1>{this.props.text} {this.state.name}</h1>
                Name: <input type="text" onChange={this.onChange} /> 
            </div>
        );
    }
});
React.render(<Greeting text="Grüzi" />, document.getElementById('container'));

React: Facts

  • Kein MV*-Framework: View Layer
  • Keine Template-Engine
  • Vergleichbar mit Angular Directives

React: Innovationen

  1. Re-Rendering statt Event- und Data-binding
  2. Virtual DOM: Schnelles Rendern
  3. Komponenten basiert

App Architektur: Flux

  • Unidirektionaler Data-Flow statt MVC
  • Viele implementierungen Verfügbar
This works especially well with React’s declarative programming style, which allows the store to send updates without specifying how to transition views between states.
